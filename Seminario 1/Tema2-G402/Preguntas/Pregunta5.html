Respuesta de la Pregunta#5:

Algunos ejemplos de los dilemas éticos a los que puedes enfrentarte como ingeniero informático

1. **Privacidad de los datos**: Imagina que estás trabajando en el desarrollo de una aplicación que recopila datos de los usuarios para mejorar la experiencia del usuario. Sin embargo, esta recopilación de datos podría invadir la privacidad de los usuarios si no se maneja correctamente. ¿Cómo equilibras la necesidad de recopilar datos con el respeto a la privacidad del usuario?

2. **Seguridad vs Usabilidad**:Estás desarrollando un sistema que maneja información sensible. Para proteger los datos, implementas medidas de seguridad estrictas que pueden hacer que el sistema sea menos fácil de usar. ¿Cómo equilibras la necesidad de seguridad con la necesidad de usabilidad?

3. **Bias en algoritmos de IA**: Estás trabajando en un algoritmo de inteligencia artificial que muestra sesgos no deseados debido a los datos de entrenamiento. ¿Cómo manejas este problema y garantizas que tu algoritmo sea justo y no discriminatorio?

4. **Obsolescencia programada**: Tu empresa quiere que desarrolles software que se vuelva obsoleto después de un cierto período de tiempo para impulsar las ventas de versiones más nuevas. ¿Es ético contribuir a la obsolescencia programada?

5. **Accesibilidad**: Estás creando una nueva aplicación web. ¿Inviertes tiempo y recursos adicionales para hacer que la aplicación sea accesible para personas con discapacidades, incluso si eso significa retrasar el lanzamiento del producto?
