Respuesta de la Pregunta#2:

La sostenibilidad del software se logra a través de varias prácticas:

Documentación: Una buena documentación ayuda a los futuros desarrolladores a entender y mantener el código.

Pruebas: Las pruebas regulares aseguran que el software sigue funcionando correctamente a medida que se realizan cambios y actualizaciones.

Arquitectura modular: Diseñar el software en módulos independientes puede facilitar la adición, eliminación o modificación de características.

Actualizaciones regulares: Mantener el software actualizado con las últimas versiones de las dependencias puede ayudar a prevenir problemas de seguridad y compatibilidad.
